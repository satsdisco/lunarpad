## Directory Purposes

**`public/`:**
- Purpose: Frontend HTML templates, stylesheets, client-side JavaScript
- Contains: Server-rendered entry points (SPA pages) + static assets
- Key files: `build.html` (main app), `style.css` (single bundled stylesheet)
- Pattern: Each page imports `style.css` and optionally `js/auth.js`

**`uploads/`:**
- Purpose: Extracted deck contents (uploaded ZIPs)
- Contains: HTML, CSS, JS, images from user presentations
- Organization: One UUID subdirectory per deck
- Security: Served via `/presentations/:id/` with path traversal protection

**`thumbnails/`:**
- Purpose: Cached WebP deck preview images
- Contains: 1280x720 WebP screenshots generated by Puppeteer
- Naming: `{deckId}.webp`
- Generated by: `generateThumbnail()` function after upload

**`avatars/`:**
- Purpose: User avatars and project banners
- Contains: PNG/JPEG images
- Naming pattern: `{userId}-avatar.{ext}`, `{projectId}-banner.{ext}`
- Served via: `/avatars/{filename}`

**`temp/`:**
- Purpose: Temporary storage during ZIP extraction and processing
- Lifecycle: Created during upload, cleaned up after extraction
- Remains: Empty between requests (not fully cleaned per request)

## Naming Conventions

**Files:**
- HTML templates: lowercase, dash-separated (`deck.html`, `admin.html`)
- CSS files: `style.css` (single file, global scope)
- JavaScript modules: `auth.js` (lowercase)
- Uploaded content: UUID-based naming (`{uuid}/` for decks, `{uuid}-avatar.png` for images)

**Directories:**
- Storage: lowercase, plural form (`uploads/`, `thumbnails/`, `avatars/`)
- Client assets: `public/` (static), `js/`, `css/` (subdirs in public)
- Database files: Root level (`deckpad.db`)

**Database Tables:**
- Lowercase, plural nouns: `users`, `decks`, `projects`, `bounties`, `events`
- Join tables: `bounty_participants`, `bounty_payments`, `project_decks`, `live_sessions`
- System tables: `_migrations`, `votes`, `comments`, `notifications`, `zaps`, `speakers`, `rsvps`

**API Routes:**
- RESTful pattern: `/api/{resource}`, `/api/{resource}/{id}`, `/api/{resource}/{id}/{action}`
- Examples: `/api/decks`, `/api/decks/:id`, `/api/bounties/:id/fund`, `/api/projects/:id/zap`
- Special: `/api/webhook/lnbits` (inbound), `/api/lightning/verify/:payment_id` (polling)

## Where to Add New Code

**New Feature (e.g., Comments API):**
- Primary code: Add route handler in `server.js` (organized by domain section)
- Database: Add table schema in `server.js` lines 43-209, add migration if post-launch
- Frontend: Add HTML page in `public/`, import shared styles and `js/auth.js`
- Example: Comments are in `comments` table (line 64), routes at lines 2489-2535

**New Component/Module:**
- Server-side utilities: Add function above route handlers in `server.js`, group by domain
- Database helpers: Add to prepared statements (`stmts` object, lines 413-445)

## Special Directories

**`uploads/`:**
- Purpose: User-generated content (extracted presentation files)
- Generated: Yes (created by file upload handler)
- Committed: No (added to .gitignore, grows with uploads)
- Access pattern: Served via `/presentations/:id/` route with path traversal protection

- Purpose: Generated preview images
- Generated: Yes (by Puppeteer after deck upload)

- Purpose: User-uploaded profile images and project banners
- Generated: No (user uploads)
- Committed: No (.gitignore)

- Purpose: Working directory for ZIP extraction during upload
- Generated: Yes (created by multer/unzipper)
- Cleanup: Files persist (no explicit cleanup between requests visible)

**Database (`deckpad.db`):**
- Purpose: Persistent state for all entities
- Generated: Yes (created on first run)
- Committed: No (.gitignore)
- Schema evolution: Via migrations table, idempotent
